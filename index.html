1. Objetivos del OVI

Objetivo general: • Modelar e Implementar una base de datos de acuerdo a un caso de estudio planteado.

Objetivo específico 1: • Identificar las diferentes entidades de acuerdo a un caso de estudio planteado. • Configurar las filas y columnas en una matriz de relaciones estableciendo las relaciones y tipos de relaciones existente entre dichas entidades encontradas. Objetivo específico 2: • Crear a partir del modelo logico de datos, un modelo físico o relacional de datos. Objetivo específico 3: • Implementar en el SGBD (Sistema Gestor de Base de Datos) de MYSQL el modelo físico de datos

2.1 UNIDAD I. Fundamentos del Modelo Lógico: (El objetivo de esta sección, es poder mostrar al usuario, un paso a paso para poder llevar a cabo la elaboración de un modelo lógico de datos, tomando como base un caso de estudio en particular, para elaborar esta parte se tendrá en cuenta la técnica Matriz de Relaciones.). En el caso propuesto se busca Identificar que entidades se encuentran involucradas en dicho escenario, también debe definir las relaciones entre las entidades encontradas, así mismo debe definir los atributos secundarios y atributos clave y plasmarlo en un modelo lógico de datos. (Diagrama E/R)

3.1 UNIDAD II. Modelo Relacional: (El objetivo de esta sección, es pasar el modelo lógico de datos a un Modelo relacional o diseño físico de datos, haciendo uso del Álgebra Relacional y Normalización de tablas del caso.) 4.1 UNIDAD III. Lenguaje estándar de consulta y herramientas: (El objetivo de esta sección, es pasar implementar el Modelo Relacional del caso planteado, en este paso, se utiliza la herramienta de implementación de SGBD llamada MYSQLWORKBENCH.)
